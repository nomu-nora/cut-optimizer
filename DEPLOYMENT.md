# デプロイガイド - 取り数最適化システム v1.0

## 📝 概要

取り数最適化システムv1.0は、ローカル環境での使用を前提としています。
このガイドでは、本番環境への展開方法を説明します。

## 🏠 ローカル実行（推奨 v1.0）

### 必要要件

- Node.js 18.0.0以上
- npm または yarn

### セットアップ手順

```bash
# 1. リポジトリをクローン
git clone <repository-url>
cd cut-optimizer

# 2. 依存関係をインストール
npm install

# 3. 開発サーバーを起動
npm run dev

# または本番ビルド版を起動
npm run build
npm run start
```

アクセス: http://localhost:3000

## 🌐 Vercelデプロイ（オプション）

将来的にクラウドデプロイが必要な場合：

### 1. Vercelアカウント準備

- https://vercel.com でアカウント作成
- GitHubリポジトリと連携

### 2. デプロイ設定

```bash
# Vercel CLIをインストール
npm i -g vercel

# プロジェクトをデプロイ
vercel
```

### 3. 環境変数（現在は不要）

v1.0では環境変数は使用していません。
将来的にAPIキーやデータベース接続が必要になった場合は、Vercelのダッシュボードから設定します。

## 📦 静的エクスポート（オプション）

完全な静的サイトとしてエクスポートすることも可能です：

```bash
# next.config.jsに以下を追加
output: 'export'

# ビルド
npm run build

# outディレクトリにエクスポートされたファイルが生成される
```

## 🔧 トラブルシューティング

### ビルドエラーが発生する場合

```bash
# キャッシュをクリア
rm -rf .next node_modules
npm install
npm run build
```

### ポート3000が使用中の場合

```bash
# 別のポートで起動
PORT=3001 npm run dev
```

## 🎯 v1.0の推奨デプロイ方法

**ローカル実行が推奨です**

理由：

- v1.0はMVP（最小実行可能製品）
- 実家での使用が主な目的
- インターネット接続不要で動作
- データは全てローカルに保存

将来のバージョンでクラウドデプロイを検討する際の要件：

- データベース統合
- ユーザー認証
- データ永続化
- マルチユーザー対応

## 📱 システム要件

### 推奨環境

- **OS**: Windows 10/11, macOS 11+, Linux
- **ブラウザ**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **画面解像度**: 1280x720以上
- **メモリ**: 4GB以上

### 動作確認済み環境

- Windows 11 + Chrome 120
- macOS Sonoma + Safari 17
- Ubuntu 22.04 + Firefox 115

## 🔒 セキュリティ考慮事項

v1.0はローカル実行のため、以下のセキュリティ対策は不要です：

- HTTPS証明書
- CORS設定
- 認証・認可
- レート制限

ただし、将来のクラウドデプロイ時には上記の対策が必要です。
